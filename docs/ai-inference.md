# <figure style="float: right; margin: 0.5ex 0.5em; line-height: 1; text-align: center; font-size: 80px">💬</figure> AI Inference

For production applications, the standard billing model is pay-as-you-go token metering, where costs scale directly with API usage. Unlike the fixed-cost [monthly subscriptions](./ai-subscriptions.md) favored by app & media developers, this approach provides maximum flexibility for applications with variable workloads. However, it places the technical burden on the developer to optimize for efficiency to control costs. On this page, we explore the ecosystem of providers specializing in token-metered inference, from large-scale cloud platforms to low-latency edge networks.

When choosing between cloud-hosted and local inference, the primary decision factor is your [hardware](./ai-hardware.md) — specifically available VRAM. Use [llmfit](./llmfit.md) to understand what can run on your machine before committing to a local-only or cloud-hybrid strategy.

## LLM Inference Specialists

As we move through 2026, the dominance of "black box" proprietary models has significantly receded, making way for a vibrant, decentralized era defined by open-weights _Mixture-of-Experts (MoE)_ architectures. However, the true democratization of these [models](./ai-models.md) hasn't occurred on local workstations, but through the rise of specialized open-source AI cloud hosting providers. These platforms—ranging from _community-driven hubs_ like 🤗 [Hugging Face](https://huggingface.co/) which serves as the central repository for the global AI community to share and collaborate on millions of open-weights models and datasets, to inference-speed innovators like Groq, Together AI, and Fireworks AI—serve as the critical bridge between raw model weights and production-ready applications. By abstracting away the brutal complexities of _GPU orchestration, high-bandwidth memory (HBM) management, and cold-start optimization,_ these providers allow developers to treat state-of-the-art intelligence as a scalable utility rather than a prohibitive infrastructure hurdle.

- [GCP Vertex](https://cloud.google.com/vertex-ai/docs/model-garden/discover-models) <img src="https://www.vectorlogo.zone/logos/google_cloud/google_cloud-icon.svg" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> is Google Cloud's enterprise-grade platform that enables developers to discover, customize, and deploy foundation models including a wide array of high-performance open-source options for production-scale AI applications.
- [Azure AI Studio](https://azure.microsoft.com/en-us/products/ai-studio) <img src="https://www.vectorlogo.zone/logos/microsoft_azure/microsoft_azure-icon.svg" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> serves as Microsoft's comprehensive unified platform that includes a massive model catalog featuring both proprietary and popular open-source models, facilitating seamless integration with the broader Azure ecosystem.
- [AWS Bedrock](https://aws.amazon.com/bedrock/) <img src="https://www.vectorlogo.zone/logos/amazon_aws/amazon_aws-icon.svg" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> is Amazon's fully managed service that offers a diverse selection of high-performing foundation models from leading AI companies, accessible via a single API to simplify building generative AI applications.
- [Cloudflare Workers AI](https://www.cloudflare.com/developer-platform/products/workers-ai/) <img src="https://www.vectorlogo.zone/logos/cloudflare/cloudflare-icon.svg" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> leverages a massive global network to provide a significant CDN and edge first-mover advantage, allowing developers to run machine learning models directly on the edge for ultra-low latency experiences.
- [DeepInfra](https://deepinfra.com/pricing) <img src="https://www.google.com/s2/favicons?sz=128&domain=deepinfra.com" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> is widely recognized in the industry for providing some of the best price-to-performance ratios available, offering a highly scalable and cost-effective infrastructure for hosting and running state-of-the-art open-source models.
- [Fireworks.ai](https://fireworks.ai/models?featured=true) <img src="https://fireworks.ai/icon0.svg" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> serves as a professional all-rounder in the inference market, providing a lightning-fast API and a robust platform that simplifies the process of deploying and fine-tuning open-weights models for various enterprise-level use cases.
- [SiliconFlow](https://www.siliconflow.com/models) <img src="https://www.google.com/s2/favicons?sz=128&domain=siliconflow.com" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> utilizes highly optimized quantization techniques like FP8 and INT4, enabling the efficient running of much larger models at the impressive speeds typically associated with significantly smaller, less capable model architectures.
- [Groq](https://groq.com/pricing) <img src="https://www.google.com/s2/favicons?sz=128&domain=groq.com" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> focuses on extreme speed using its proprietary Language Processing Unit (LPU) technology, specifically designed to provide the low-latency performance required for real-time agentic workflows and highly interactive AI-driven application experiences.
- [Together AI](https://www.together.ai/models) <img src="https://www.google.com/s2/favicons?sz=128&domain=together.ai" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> is known for rapidly providing access to fast new model versions and maintaining a good variety of open-weights models, supporting researchers and developers with a powerful, scalable research-friendly inference environment.
- [Nebius AI](https://nebius.com/token-factory/prices) <img src="https://www.google.com/s2/favicons?sz=128&domain=nebius.com" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> primarily serves the European market by providing high-performance GPU clusters and a managed inference platform that complies with regional data sovereignty requirements while offering competitive performance for modern AI workloads.

Selecting a cloud hosting provider is no longer a simple race to the bottom on per-token pricing, but a strategic alignment with the specific requirements of your AI architecture. The market has bifurcated into specialized niches: those who prioritize the raw, sub-millisecond throughput of custom LPU hardware, those who offer the deepest model catalogs and fine-tuning flexibility, and enterprise-grade players like SiliconFlow that focus on sovereign, compliance-heavy private clouds. As agentic workflows and real-time multimodal systems become the standard, the competitive edge has shifted toward the "inference stack"—the proprietary software layers that optimize how [models](./ai-models.md) actually run on silicon. In this landscape, your hosting partner is far more than a utility vendor; they are the high-performance engine room that determines the latency, reliability, and ultimate viability of your AI strategy.

## Inference on the Edge

In the cloud, providers leverage Content Delivery Networks (CDN's) to serve low-latency inference at the edge, enabling real-time AI features in web applications with global reach.
The major hyperscalers (AWS, Google Cloud, and Azure) have shifted their architecture to compete directly with CDN's by offering "Edge Functions" and "Serverless GPU" services. While Vercel focuses on developer experience and Cloudflare on a massive global network of small points of presence, the hyperscalers leverage their massive GPU "Regional Edges" to handle heavier AI workloads.

- [Cloudflare Workers AI](https://developers.cloudflare.com/workers-ai/) <img src="https://www.vectorlogo.zone/logos/cloudflare/cloudflare-icon.svg" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> allows developers to run machine learning inference directly on Cloudflare's global network, ensuring that AI-powered applications remain highly responsive by processing data closer to the end-users.
- [Vercel AI SDK](https://ai-sdk.dev/docs/introduction) <img src="https://www.vectorlogo.zone/logos/vercel/vercel-icon.svg" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> provides a comprehensive toolkit for building AI-driven user interfaces, offering seamless integration with various model providers and edge functions to streamline the development of modern high-performance web applications.
- Amazon <img src="https://www.vectorlogo.zone/logos/amazon/amazon-icon.svg" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> provides a robust suite of cloud services that support globally distributed AI applications, leveraging their extensive infrastructure to offer scalable and reliable solutions for developers building complex high-traffic systems.
  - [AWS Lambda Edge](https://aws.amazon.com/lambda/edge/): <img src="https://www.vectorlogo.zone/logos/amazon_aws/amazon_aws-icon.svg" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> is the direct competitor to Cloudflare Workers, allowing you to run Node.js or Python code at AWS CloudFront locations globally to handle AI tasks like request routing and prompt sanitization efficiently.
- Google <img src="https://www.vectorlogo.zone/logos/google/google-icon.svg" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> continues to lead in AI infrastructure, offering a wide range of powerful tools and services that enable developers to build, train, and deploy sophisticated machine learning models at a global scale.
  - [GCP Cloud Functions (2nd Gen)](https://cloud.google.com/blog/products/serverless/cloud-functions-2nd-generation-now-generally-available): <img src="https://www.vectorlogo.zone/logos/google_cloud/google_cloud-icon.svg" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> competes with Cloudflare Workers by utilizing advanced "Min-instances" technology to eliminate cold starts, making it a viable solution for real-time AI API middleware and other latency-sensitive applications.
- Microsoft Azure <img src="https://www.vectorlogo.zone/logos/microsoft_azure/microsoft_azure-icon.svg" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> offers a comprehensive set of cloud services designed to support every stage of the AI development lifecycle, providing developers with the tools and infrastructure needed to build and scale intelligent applications.
  - [Azure Static Web Apps](https://learn.microsoft.com/en-us/azure/static-web-apps/overview) (SWA): <img src="https://www.vectorlogo.zone/logos/microsoft_azure/microsoft_azure-icon.svg" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> is the direct competitor to Vercel for frontend-heavy AI apps, integrating seamlessly with Azure Functions for back-end AI logic and providing "First-class" support for the Azure OpenAI Service.
  - [Azure Functions (Flex Consumption)](https://learn.microsoft.com/en-us/azure/azure-functions/flex-consumption-plan): <img src="https://www.vectorlogo.zone/logos/microsoft_azure/microsoft_azure-icon.svg" align="right" style="width: 18px; margin-top: 0.1ex; border-left: 0.5em solid transparent;"> represents a 2026 evolution in serverless computing, allowing for high-concurrency AI processing and providing guaranteed memory and CPU resources for token-heavy large language model streaming and other demanding workloads.
